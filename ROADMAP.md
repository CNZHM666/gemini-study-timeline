# Roadmap

This roadmap is intentionally practical. The goal is not to turn Hajimi Timeline into a huge all-in-one product, but to make it feel polished, reliable, and worth using every day.

## Current Priorities

### 1. Workflow Clarity

- make the study queue and workspace relationship more obvious
- reduce duplicated prompts and visual noise
- keep actions predictable and easy to learn

### 2. Notes And Review Experience

- make note taking feel lighter and more focused
- improve the save, backup, and review flow
- keep note history useful without making the panel heavy

### 3. Long-Conversation Performance

- reduce rendering pressure on very long Gemini chats
- keep scanning and syncing stable during long sessions
- make low-footprint mode more dependable

### 4. Public Project Readiness

- improve screenshots, GIF demos, and documentation
- make contribution and release flow more standard
- keep repository structure clean and easy to understand

## Near-Term Ideas

- clearer review wording and review state naming
- better onboarding hints for first-time users
- more consistent empty states across panels
- cleaner GitHub-facing demo materials

## Things We Intentionally Keep Tight

- local-first storage model
- minimal dependency surface
- side-panel-first workflow
- focused learning use case instead of generic chat tooling

## Not A Priority Right Now

- turning this into a cloud-sync product
- adding many unrelated productivity features
- making the UI more complex just to expose more controls
- supporting many non-Gemini sites before the Gemini workflow is fully polished

## Contribution Hint

If you want to contribute, the highest-value work is usually:

- removing friction from the learning workflow
- improving clarity in the workspace UI
- fixing long-chat performance issues
- improving backup reliability and local persistence confidence
