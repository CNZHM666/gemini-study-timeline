## Public Launch Checklist

- Confirm `README.md` is readable without prior context.
- Confirm `LICENSE` is present.
- Confirm `CHANGELOG.md` is current enough for first public visitors.
- Confirm `CONTRIBUTING.md` and `ROADMAP.md` match the current direction.
- Confirm `npm run check` passes.
- Confirm `powershell -ExecutionPolicy Bypass -File .\scripts\release.ps1 -DryRun` passes.
- Confirm there is no private or accidental data in screenshots.
- Confirm `dist/` and zip outputs are ignored unless intentionally published.

## First Public Impression Checklist

- Repository avatar or owner profile looks complete.
- Description and topics are filled in.
- README top section is visible without too much scrolling.
- At least one screenshot or GIF is added soon after launch.
- First release notes are published so visitors see active maintenance.
